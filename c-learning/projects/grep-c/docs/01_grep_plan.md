# Build Your Own Grep (C语言版) - 实施计划

这是一个非常经典的项目，非常适合学习 C 语言的核心概念（指针、内存管理、文件操作）。我们将一步步来实现。

## 目标
构建一个简化版的 `grep` 工具，命令格式如下：
```bash
./mygrep <搜索词> <文件名>
```

## 阶段规划

### 第一阶段：不仅是 Hello World (基础搭建)
我们将建立项目结构，并学习如何编译 C 程序。
*   **目标**: 能够运行 `./mygrep` 并打印 "Grep starts..."
*   **知识点**: `main` 函数，`printf`，GCC 编译命令，Makefile。

### 第二阶段：处理命令行参数 (Inputs)
程序需要知道用户想搜什么，以及去哪里搜。
*   **目标**: 运行 `./mygrep hello test.txt`，程序能打印出：
    > Pattern: hello
    > File: test.txt
*   **知识点**: `argc` (参数个数), `argv` (参数列表), 字符串数组。

### 第三阶段：读取文件内容 (File I/O)
学习如何打开文件并逐行读取内容。
*   **目标**: 打印出 `test.txt` 的每一行内容。
*   **知识点**: `fopen` (打开), `getline` (读取行), `fclose` (关闭), 指针 `FILE*`。
*   **难点**: 理解 C 语言中的缓冲与内存管理。

### 第四阶段：核心搜索逻辑 (Logic)
在读取的每一行中查找关键词。
*   **目标**: 只打印包含 "hello" 的行。
*   **知识点**: `strstr` (查找子串), 条件判断 `if`。

### 第五阶段：错误处理 (Robustness)
如果文件不存在怎么办？
*   **目标**: 优雅地提示 "Error: Could not open file"。
*   **知识点**: 返回值检查，`stderr` (标准错误输出)。

## 验证计划
我们将创建一个测试文件 `poem.txt`，用它来验证我们的程序是否能像真正的 `grep` 一样工作。
```bash
# 预期测试命令
./mygrep "rose" poem.txt
```
