# 🎓 C 语言 Grep 项目总结

恭喜！您已经从零开始构建了一个功能完整的命令行工具。

## 🌟 核心成果
您亲手实现了通过 `Clang` 编译的 C 程序 `mygrep`，它支持：
1.  **命令行参数解析** (`argc`, `argv`)
2.  **文件读取** (`fopen`, `fgets`, `fclose`)
3.  **字符串搜索** (`strstr`, `strcasestr`)
4.  **功能增强**：行号显示 + 忽略大小写 (`-i`)
5.  **代码重构**：函数封装 (`grep_file`)

## 🧠 关键知识点回顾

### 1. 指针无处不在
*   `char *argv[]`: 字符串数组（指针的数组）。
*   `FILE *file`: 指向文件结构的指针。
*   `strstr(...)`: 返回找到位置的指针。

### 2. 内存与缓冲区
我们使用了 `char line[256]` 作为缓冲区。
*   **思考**: 如果一行超过 256 个字符会发生什么？(提示：`fgets` 只会读前 255 个字符，剩下的下次循环读)。

### 3. Linux/GNU 扩展
为了实现 `-i`，我们使用了非标准的 `strcasestr`，这需要 `#define _GNU_SOURCE`。这是系统编程中常见的情况：在标准 (ANSI C) 和实用 (POSIX/GNU) 之间权衡。

虽然程序能跑了，但作为一个“工程”，它还缺最后一块拼图：**Makefile**。

*   **建议**: 我们可以写一个 `Makefile`，以后只需输入 `make` 就能自动编译。
